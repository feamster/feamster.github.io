Studying Routing Configuration Errors in Large ISPs

Sponsor

Prof. Nick Feamster
feamster@cc.gatech.edu
Area Networking

Problem

Internet routing, the process by which hosts construct paths to destinations, must be highly configurable to provide scalability and allow network operators to express flexible business policies. In some sense, configuring a network of routers to run a routing protocol is like writing a large distributed program. This configurability provides flexibility, but also introduces complexity that can lead to errors.

Recent work has shown that analyzing the static configuration files is surprisingly useful for detecting errors and predicting the dynamic behavior of the Internet's routing protocol, the Border Gateway Protocol (BGP). This project involves further exploring the types of errors that static configuration analysis can detect. For example, you could investigate errors caused by the interactions between a network's BGP and internal routing configurations (see the background papers for more details on this problem). You could also analyze some other aspect of the network configuration (e.g., a network's VPN configuration). In any case, you'll be finding errors in the configurations of operational networks.

Background

You'll likely find these papers useful. The first one, on rcc, is particularly relevant, as it describes the tool that performs static analysis on BGP configurations.

  • N. Feamster and H. Balakrishnan
    Detecting BGP Configuration Faults with Static Analysis
    Usenix NSDI, May 2005.

  • T. Griffin and G. Wilfong
    On the Correctness of iBGP Configuration
    ACM SIGCOMM, August 2002.

  • N. Feamster and J. Rexford
    A Model of BGP Routing for Network Engineering
    ACM SIGMETRICS, June 2004.
You'll want to have a look at the rcc code, available here. You'll also want to contact me to gain access to the router configuration files on which you can test your code (and hopefully find some errors!).

Deliverables

Your additions to the rcc codebase, and a short (less than 5-page) writeup describing your analysis of errors in real-world router configurations.

Evaluation

Based on the quality of your analysis. Finding errors makes for excellent results, but developing sound checks is more important!
